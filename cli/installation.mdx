## Demo preview download

Version `0.2.0-preview.4` is hosted in the D-Tools demo Azure environment. It is
intended for evaluation against demo accounts and is not a stable customer release.

<CodeGroup>

```bash macOS / Linux
curl -fsSL https://dtclouddemostorage.blob.core.windows.net/dtcloud-cli-preview/install/dtcloud.sh | bash
```

```powershell Windows
irm https://dtclouddemostorage.blob.core.windows.net/dtcloud-cli-preview/install/dtcloud.ps1 | iex
```

</CodeGroup>

The Unix installer pins the demo release public key, verifies the RSA-PSS signed
manifest and artifact SHA-256, installs under `~/.local/bin`, and writes a bound
receipt under `~/.dtcloud/install-receipt.json`.

<Warning>
  The preview PowerShell installer verifies the artifact SHA-256 but does not yet
  verify the RSA manifest signature. Use it only for demo evaluation. Signed MSI
  and production PowerShell installation remain blocked from stable promotion.
</Warning>

Direct downloads are also available:

- [macOS Apple silicon](https://dtclouddemostorage.blob.core.windows.net/dtcloud-cli-preview/artifacts/0.2.0-preview.4/dtcloud_0.2.0-preview.4_darwin_arm64)
- [macOS Intel](https://dtclouddemostorage.blob.core.windows.net/dtcloud-cli-preview/artifacts/0.2.0-preview.4/dtcloud_0.2.0-preview.4_darwin_amd64)
- [Linux x64](https://dtclouddemostorage.blob.core.windows.net/dtcloud-cli-preview/artifacts/0.2.0-preview.4/dtcloud_0.2.0-preview.4_linux_amd64)
- [Linux ARM64](https://dtclouddemostorage.blob.core.windows.net/dtcloud-cli-preview/artifacts/0.2.0-preview.4/dtcloud_0.2.0-preview.4_linux_arm64)
- [Windows x64](https://dtclouddemostorage.blob.core.windows.net/dtcloud-cli-preview/artifacts/0.2.0-preview.4/dtcloud_0.2.0-preview.4_windows_amd64.exe)
- [Windows ARM64](https://dtclouddemostorage.blob.core.windows.net/dtcloud-cli-preview/artifacts/0.2.0-preview.4/dtcloud_0.2.0-preview.4_windows_arm64.exe)

Set the preview release origin before checking for updates or installing Agent Packs:

<CodeGroup>

```bash macOS / Linux
export DTCLOUD_RELEASE_BASE_URL=https://dtclouddemostorage.blob.core.windows.net/dtcloud-cli-preview/api/releases
dtcloud update-check --channel preview --json
```

```powershell Windows
$env:DTCLOUD_RELEASE_BASE_URL = 'https://dtclouddemostorage.blob.core.windows.net/dtcloud-cli-preview/api/releases'
dtcloud update-check --channel preview --json
```

</CodeGroup>

Verify the installed surface:

```bash
dtcloud version --json
dtcloud self update --channel preview --dry-run --json
```

## Install from a build artifact

Download the artifact for your platform from the `dtcloud-cli-release` Azure DevOps pipeline.
The pipeline produces the following artifact families for each release:

| Format | Platforms |
| --- | --- |
| `.zip` | Windows (`amd64`, `arm64`) |
| `.tar.gz` | Linux (`amd64`, `arm64`), macOS (`amd64`, `arm64`) |
| `.deb` | Linux (`amd64`, `arm64`) |
| `.rpm` | Linux (`amd64`, `arm64`) |
| `.msi` | Windows installer |
| `.pkg` | macOS installer |

<Note>
  Development and PR artifacts are unsigned. Tagged release artifacts are signed. The macOS
  `.pkg` is signed and notarized only for tagged releases.
</Note>

After extracting the archive or running the installer, verify the binary:

<CodeGroup>

```powershell Windows
dtcloud.exe --version
```

```bash macOS / Linux
dtcloud --version
```

</CodeGroup>

The macOS `.pkg` installs `dtcloud` into `/usr/local/bin/dtcloud`. The Windows MSI installs
`dtcloud.exe` under `Program Files\DToolsCloudCLI` and adds that directory to the machine
`PATH`.

## First run

<Steps>
  <Step title="Sign in">
    Start the device-code authentication flow:

    ```bash
    dtcloud auth login
    ```

    The CLI prints a verification URL and a short code. Open the URL in any browser, enter the
    code, and complete the Entra External ID sign-in. The resulting MSAL token is cached at
    `~/.dtcloud/token.json` with `0600` permissions.
  </Step>

  <Step title="Check auth status">
    Confirm the token was cached successfully:

    ```bash
    dtcloud auth status --output yaml
    ```
  </Step>

  <Step title="List accounts">
    See all D-Tools Cloud accounts visible to your user:

    ```bash
    dtcloud accounts list
    ```
  </Step>

  <Step title="Select an account">
    Persist the account you want to use for subsequent commands:

    ```bash
    dtcloud accounts use 12345
    ```

    All account-scoped commands will include `X-DTools-AccountId: 12345` automatically. Use
    `--account <id>` for a one-off override without changing the persisted selection.
  </Step>
</Steps>

## Selecting a profile

The CLI ships with a `dev` profile configured by default. To point it at a different
environment, create or update a profile:

```bash
dtcloud profile set staging \
  --base-url https://stgapi.d-tools.cloud \
  --authority https://example.ciamlogin.com/<tenant-id> \
  --client-id <public-client-id> \
  --scope api://<api-client-id>/access_as_user \
  --default
```

The production profile is configured by default. Use it explicitly unless you have made
it your default profile:

```bash
dtcloud --profile prod auth login
dtcloud --profile prod accounts list
```

If you already created `~/.dtcloud/config.yaml` before a default profile was updated,
reset it:

```bash
dtcloud profile set dev --base-url https://dtoolsdevapi.azure-api.net/api/v2
dtcloud profile set prod \
  --base-url https://api.d-tools.cloud/api/v2 \
  --authority https://fb1ad9bb-9bd2-45fe-a065-c8a0370c2913.ciamlogin.com/fb1ad9bb-9bd2-45fe-a065-c8a0370c2913 \
  --client-id 6ccf1cdf-d7f7-4aa4-a292-16ac4c1d6504 \
  --scope api://0d607d3e-ee5d-4992-a722-9f5be9c67a17/access_as_user
```

Use a profile for a single command without changing the default:

```bash
dtcloud --profile staging clients list
```

## Build from source

Use this path when you need a specific target platform, CPU architecture, or package format,
or when no pre-built artifact is available.

### Prerequisites

- **Go** `1.26.3` or the version declared by `go.mod` _(verify this — check `Tools/dtcloud-cli/go.mod` for the current requirement)_.
- **Git** (required for version metadata).
- **Bash** on macOS/Linux or **PowerShell** on Windows.

Additional tools for release-style packages:

- **GoReleaser v2** and **Syft** — required for archives, debs, rpms, and SBOM generation.
- **WiX** on a Windows host — required for `.msi` builds.
- **`pkgbuild`** on a macOS host — required for `.pkg` builds (signing and notarization require Apple signing assets and are only performed in release CI).

### Local host build

Run the repo-provided validation and build script. These scripts run formatting checks,
`go test ./...`, `go vet ./...`, and produce a local binary under `Tools/dtcloud-cli/dist/`.

<CodeGroup>

```bash macOS / Linux
scripts/build.sh
```

```powershell Windows
scripts/build.ps1
```

</CodeGroup>

### Build a single target binary

<CodeGroup>

```bash macOS / Linux
version="0.1.0-dev"
commit="$(git rev-parse --short HEAD)"
date="$(date -u +%Y-%m-%dT%H:%M:%SZ)"

GOOS=linux GOARCH=amd64 CGO_ENABLED=0 \
  go build -trimpath \
  -ldflags "-s -w \
    -X dtools.cloud/dtcloud-cli/internal/buildinfo.Version=${version} \
    -X dtools.cloud/dtcloud-cli/internal/buildinfo.Commit=${commit} \
    -X dtools.cloud/dtcloud-cli/internal/buildinfo.Date=${date}" \
  -o dist/dtcloud-linux-amd64 \
  ./cmd/dtcloud
```

```powershell Windows
$version = "0.1.0-dev"
$commit = (git rev-parse --short HEAD)
$date = (Get-Date).ToUniversalTime().ToString("yyyy-MM-ddTHH:mm:ssZ")

$env:GOOS = "windows"
$env:GOARCH = "amd64"
$env:CGO_ENABLED = "0"
go build -trimpath `
  -ldflags "-s -w -X dtools.cloud/dtcloud-cli/internal/buildinfo.Version=$version -X dtools.cloud/dtcloud-cli/internal/buildinfo.Commit=$commit -X dtools.cloud/dtcloud-cli/internal/buildinfo.Date=$date" `
  -o dist/dtcloud-windows-amd64.exe `
  ./cmd/dtcloud
```

</CodeGroup>

Change `GOOS`, `GOARCH`, and the output file name for your target. Use an `.exe` suffix when
`GOOS=windows`.

### Build all packages (archives, debs, rpms)

<CodeGroup>

```bash macOS / Linux
scripts/package.sh
```

```powershell Windows
scripts/package.ps1
```

</CodeGroup>

Or run GoReleaser directly:

```bash
GORELEASER_CURRENT_TAG=v0.1.0 goreleaser release --snapshot --clean
```

Artifacts are written to `Tools/dtcloud-cli/dist/`. The Windows package script also creates
a local MSI when WiX is on `PATH`; the macOS package script also creates an unsigned `.pkg`
when `pkgbuild` is available.

### Verify the build

```bash
dist/dtcloud --version
dist/dtcloud health --output json
```

### Common build issues

| Symptom | Likely cause | Fix |
| --- | --- | --- |
| `goreleaser: command not found` | GoReleaser not installed | Install GoReleaser v2 and retry. |
| `syft: command not found` | SBOM tool missing | Install Syft, or use plain `go build` for binaries only. |
| MSI build skipped | WiX missing or not on `PATH` | Install/update WiX on a Windows host. |
| `.pkg` build skipped | Not on macOS, or `pkgbuild` unavailable | Build `.pkg` on macOS. |
| Windows binary missing `.exe` | Output name omitted extension | Use `.exe` suffix when `GOOS=windows`. |
| Binary runs but auth fails | Local profile or token issue | Run `dtcloud auth status`, verify `~/.dtcloud/config.yaml`. |
